# WITHIN Rai Prototype

RAG pipeline demo showing how call transcripts and Slack channel data are added as knowledge sources to an internal chatbot, with hybrid search, client-level filtering, and live ingestion.

## Setup

### 1. Create and activate a virtual environment

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

> **Note:** `FlagEmbedding` with `BAAI/bge-m3` downloads ~2.3 GB on first run. The model is cached locally after the first download.

### 2. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` with your credentials:

| Variable | Where to get it |
|---|---|
| `QDRANT_URL` | Qdrant Cloud dashboard → Cluster → Connect |
| `QDRANT_API_KEY` | Qdrant Cloud dashboard → API Keys |
| `ANTHROPIC_API_KEY` | console.anthropic.com |

Create a free Qdrant Cloud account at [cloud.qdrant.io](https://cloud.qdrant.io) if you haven't already.

### 3. Generate synthetic data

```bash
python data/synthetic/generate_data.py
```

This writes `data/synthetic/transcripts.json` and `data/synthetic/slack_messages.json`.

### 4. Run the API

```bash
uvicorn app.main:app --reload
```

The API starts at `http://localhost:8000`. On startup it creates the `rai_knowledge_base` collection in Qdrant if it doesn't exist.

### 5. Run the frontend

In a second terminal:

```bash
streamlit run frontend/app.py
```

Opens at `http://localhost:8501`.

### 6. Seed data

Click **"Seed sample data"** in the Streamlit sidebar. This ingests all synthetic transcripts and Slack messages. Takes ~2 minutes (embedding the chunks locally).

## API Reference

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Health check + Qdrant status |
| `POST` | `/ingest/transcript` | Ingest a call transcript |
| `POST` | `/ingest/slack` | Ingest a batch of Slack messages |
| `POST` | `/chat` | Query the RAG pipeline |

### Example: ingest a new transcript

```bash
curl -X POST http://localhost:8000/ingest/transcript \
  -H "Content-Type: application/json" \
  -d '{
    "transcript_id": "t_demo_001",
    "client": "Casper",
    "meeting_name": "Ad Hoc Strategy Call",
    "date": "2026-04-09",
    "participants": ["Sarah (AM)", "Jake (Client)"],
    "text": "Sarah: Jake, just a quick note — we saw a 15% CTR lift overnight after the creative refresh."
  }'
```

### Example: query

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "How did Casper Facebook campaign perform?", "client": "Casper"}'
```

## Deploy to Render

1. Push to a GitHub repo
2. Go to [render.com](https://render.com) → New → Blueprint
3. Connect the repo — Render will detect `render.yaml` and create two services:
   - `rai-api` (FastAPI)
   - `rai-frontend` (Streamlit)
4. Set the env vars in the Render dashboard for `rai-api`: `QDRANT_URL`, `QDRANT_API_KEY`, `ANTHROPIC_API_KEY`
5. Set `API_URL` for `rai-frontend` to the deployed URL of `rai-api` (e.g. `https://rai-api.onrender.com`)
6. After both services are live, seed data via the Streamlit UI

## Architecture

```
Streamlit UI  →  FastAPI  →  Ingestion (chunk + embed)  →  Qdrant Cloud
                          →  RAG (hybrid search + Claude generation)
```

- **Embeddings:** BGE-M3 (BAAI/bge-m3) — produces dense (1024-dim) + sparse vectors from one model
- **Search:** Qdrant RRF fusion of dense + sparse prefetch queries
- **Filtering:** `client` field filter enforced server-side on every Qdrant query — no cross-client leakage
- **Recency:** Post-retrieval score multiplier favors documents from the last 90 days
- **Citations:** Sources pulled from Qdrant payload metadata, not generated by Claude
